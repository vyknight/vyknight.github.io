This website is made using only HTML and CSS with Bootstrap. It's currently hosted on Github Pages for free.

I chose not to use any JavaScript frameworks, or JavaScript at all because I think that would be over-engineering for a simple portfolio site such as this, which has 4 pages, some text, and a picture of me. As such, the code for this webpage is only about 7 kb! I'm really happy about this because I really enjoy optimizing my projects to use as little resources as possible.

The design choices I've made for this website comes in part from my nostalgia for the Web 1.0 days, when everything was more or less laid bare.

This is the first time I've taken a website from design to deployment entirely on my own. Not the worst eh?

PokerGPT is a LLM powered poker coach, our goal is make poker coaching accessible and affordable for everyone.

Since this project is very experimental, I can't go into all the details. But I'm able to divulge that this project uses React with NextJS on the frontend, Sass and a headless component library for styling, Prisma ORM for accessing our PostgreSQL database, and a FastAPI backend.

I helped to draft the architecture, and as the main backend developer I'm responsible for running AI experiments.

Earth is an all-in-one platform for running hackathons, its goals are to address the shortcomings we find in popular hackathon mangement software such as Agorize.

Our tech stack consists of TypeScript, React, NextJS, a CSS component library, and a PostgreSQL database.

I own the judging and user directory feature, and I'm typically the first person people reach out to when they have questions regarding API and database access. Over my internship I've implemented pagination, connection pooling, and fuzzy search. I'm also the primary driver for our adoption of Prisma ORM.

Although Earth have not yet been fully released, it was widely appraised when we beta'd the program during the French multinational Capgemini's internal GenAI Hackathon. During which it was used by 1062 participants worldwide

In my first software engineering course our project was an ASCII art trading hub. It was during the time when NFTs were still a hot commodity, and we decided to make a mini version of OpenSeas

Our techstack consists almost entirely of Java, with persistent storage handled by a MongoDB database. We also attempted to port the program to Android

We were graded on how closely we followed best software engineering principles like SOLID and clean architecture. But our goal with the project was to create a living system or game in which users can submit (mint) ASCII art they've found on the internet that wasn't yet present in the system, the art then could be traded with other players. There was also supposed to be a "house" player in which the game system itself will purchase and sell art at certain intervals to discreetly manipulate the market.

I was the team lead, my responsibilities consists of drafting the architecture and delegating team members onto tasks. Not to say that I was just managing, I wrote most of the code for wallets and trading, as well as all the documentation.

This project seeks to analyze what topics around Covid suffers the most sensationalism

This project is my first university CS project, as such it uses Python exclusively with a few libraries for scraping, data analysis, and data visualization

I was mostly responsible for scraping the data to be analyzed, I was able to come up with a script to download tens of thousands of tweets at a time into a CSV file, filterable by keywords or region

The scraped data is then scored using the Natural Language Tool Kit, and then graphed with plotly and pandas.
